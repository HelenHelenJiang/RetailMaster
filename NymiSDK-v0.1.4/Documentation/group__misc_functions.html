NclBool nclNotify ( int  nymiHandle,
NclBool  good 
)

The LED lights and the vibration motor of the Nymi are mainly used for built-in functions such as reporting the state of the Nymi. As a developer, you can send simple notifications to the user to report either a "good" state or a "bad" state. A "good" notification makes the Nymi vibrate and light up all its LEDs. A "bad" notification makes the Nymi vibrate and blink all its LEDs.

Precondition
NCL is initialized. The specified Nymi is validated and has a free command channel.
Postcondition
The Nymi's command channel is taken. An NCL_EVENT_NOTIFIED will be triggered when the command channel is free. The Nymi will notify its user.
Parameters
[in]nymiHandleThe handle of the Nymi whose user you want to notify.
[in]goodNCL_TRUE to send a "good" notification, NCL_FALSE to send a "bad" notification.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.
NclBool nclGetRssi ( int  nymiHandle)

RSSI can be used to estimate the distance between the Nymi and the device that the Nymi-Enabled NEA is running on. Smaller RSSI indicates larger distance. Due to noise in RSSI values, you may wish to collect 5-15 samples in sequence and use the mean or median value. This function is not guaranteed to produce a corresponding event, and there are a few reasons why. RSSI is present in every BLE packet, this operation simply captures the RSSI on the next packet and sends it along. As such, this operation may happen while the command channel is taken. This also means that a failure to trigger an RSSI event means greater failures are happening. You may also call this function more than once before an RSSI event is triggered.

Precondition
NCL is initialized. The specified Nymi is connected.
Postcondition
An NCL_EVENT_RSSI may be triggered.
Parameters
[in]nymiHandleThe handle of the Nymi you want to get RSSI from.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.
NclBool nclGetFirmwareVersion ( int  nymiHandle)

Asks the Nymi to send its firmware version.

Precondition
NCL is initialized. The specified Nymi is validated and has a free command channel.
Postcondition
The Nymi's command channel is taken. An NCL_EVENT_FIRMWARE_VERSION will be triggered, at which point the command channel is free.
Parameters
[in]nymiHandleThe handle of the Nymi whose firmware version you want.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.
