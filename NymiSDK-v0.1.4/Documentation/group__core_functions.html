NclBool nclInit ( NclCallback  callback,
void *  userData,
const char *  name,
NclMode  mode,
FILE *  errorStream 
)

Initialize the library and start BLE initialization. If this function fails, there is no guarantee of recoverability. This is because failure will be caused by something beyond the scope of the program, for example lack of appropriate hardware.

Precondition
nclInit has never been called, or nclFinish has just been called.
Postcondition
On successful initiation, an NCL_EVENT_INIT will be triggered. Until nclFinish is called, NCL_EVENT_ERROR will be triggered if an error occurs.
Parameters
[in]callbackCallback function that at least handles NCL_EVENT_INIT. This callback is added as if nclAddBehavior(callback, userData, NCL_EVENT_ANY, NCL_NYMI_HANDLE_ANY) were called.
[in]userDataAny custom data you would like to pass into the callback function when it is triggered. You can pass NULL if you don't have any custom data.
[in]nameThe name of your NEA. (A user will be able to see a list of connected apps from the Nymi Companion App). 18 characters or less, not including null terminator.
[in]modeA bit mask representing the desired mode. See NclMode.
[in]errorStreamError messages are sent here. Typically this would be set to stderr, or a file stream. Pass NULL if you don't want errors printed. You should habitually check the error stream if anything goes wrong.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.
NclBool nclFinish ( )

Terminate the library. If this function fails, there is no guarantee of recoverability, so a program should exit. This is because failure will be caused by something beyond the scope of the program. Do not call this function from within an NCL callback.

Precondition
NCL is initialized.
Postcondition
All resources NCL took are released and nclInit may be called again.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.
NclBool nclUpdate ( unsigned  timeout)

In synchronous mode (set by nclInit), this function must be called at regular intervals to send accumulated events to callbacks. This function returns NCL_FALSE in asynchronous mode. Learn more in the Concurrency Modes section.

Precondition
NCL is in synchronous mode.
Postcondition
Queued events will have triggered appropriate callbacks.
Parameters
[in]timeoutMaximum time in milliseconds to wait for events.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.
NclBool nclSetIpAndPort ( const char *  ip,
int  port 
)

Some flavors of NCL use an ecodaemon to talk to the Nymi ecosystem. The ecodaemon is communicated with over the network. There is a default IP address and port number at which to find the ecodaemon, so in most cases you don't need to worry about this function. However, if you are using an ecodaemon, and if you need it to be at a custom IP address or port, you can call this function to tell the NEA where to find the ecodaemon.

Precondition
You are using an ecodaemon NCL flavor. NCL is not initialized.
Postcondition
The IP address and port to find the ecodaemon during nclInit will be set.
Parameters
[in]ipNull-terminated string representation of the IP address. By default, this is "127.0.0.1".
[in]portThe port number. By default, this is 9089.
Returns
NCL_TRUE On success.
NCL_FALSE On failure.
NclBool nclAddBehavior ( NclCallback  callback,
void *  userData,
NclEventType  eventType,
int  nymiHandle 
)

A behavior consists of: the callback function to use, custom data for the callback, the event type that should trigger the callback, and the handle of the Nymi to listen to. When an event is triggered, the NCL checks an internal list of behaviors and calls all callbacks whose event type and Nymi handle match the event, in the order they were added. This function allows you to add behaviors. You cannot add the same behavior twice (i.e. same callback function, user data, event type, and Nymi handle). You can learn about the event-driven nature of the NCL in Working with the Nymi Communication Library.

Precondition
NCL is initialized.
Postcondition
On success, the specified behavior is added.
Parameters
[in]callbackA callback function.
[in]userDataAny custom data you would like to pass into the callback function when it is triggered. Pass in NULL if you don't have any custom data.
[in]eventTypeThe type of event you'd like the callback to receive. Pass in NCL_EVENT_ANY to listen for all events.
[in]nymiHandleThe integer handle of the Nymi you'd like the callback to respond to. Pass in NCL_NYMI_HANDLE_ANY to listen to all Nymis and events without Nymi handles.
Returns
NCL_TRUE On success.
NCL_FALSE On failure.
NclBool nclRemoveBehavior ( NclCallback  callback,
void *  userData,
NclEventType  eventType,
int  nymiHandle 
)

This function allows you to remove behaviors that were added using nclAddBehavior or nclInit, by specifying their parameters when added. You can learn about the event-driven nature of the NCL in Working with the Nymi Communication Library. Note that this function removes one behavior at a time, so you cannot remove multiple behaviors with NCL_EVENT_ANY or NCL_NYMI_HANDLE_ANY.

Precondition
NCL is initialized.
Postcondition
On success, the specified behavior is removed.
Parameters
[in]callbackCallback of the previously specified behavior to remove.
[in]userDataUser data of the previously specified behavior to remove.
[in]eventTypeEvent type of the previously specified behavior to remove.
[in]nymiHandleNymi handle of the previously specified behavior to remove.
Returns
NCL_TRUE On success.
NCL_FALSE On failure.
NclBool nclLockErrorStream ( )

In asynchronous mode (set by nclInit), the error stream may be written to at any point. Say you provided to nclInit a stream that you also want to be able to write to. If you try to write to the stream without locking it, the NCL could interrupt your write with its own write, creating a gibberish stream. This function allows you to lock it (i.e. prevent additional errors from being written). Make sure you unlock the error stream with nclUnlockErrorStream after you are done with it. This function returns NCL_FALSE in synchronous mode.

Precondition
NCL is in asynchronous mode.
Postcondition
The NCL error stream is locked.
Returns
NCL_TRUE On success.
NCL_FALSE On failure.
NclBool nclUnlockErrorStream ( )

After calling nclLockErrorStream, this function allows you to unlock the error stream so that more errors can be written to it.

Precondition
NCL is in asynchronous mode.
Postcondition
The NCL error stream is unlocked.
Returns
NCL_TRUE On success.
NCL_FALSE On failure.
NclErrorCode nclGetErrorCode ( )

Error codes are meant to give enough information to differentiate between which possible programmatic responses to failures should happen. When a failure happens, if the error code is currently clear, the error code is set. This function returns the current error code and then clears it. A failure is typically indicated by an NCL function returning NCL_FALSE, or an NCL_EVENT_ERROR; This function gives the same concurrency guarantees that other NCL functions do, but you still need to analyze your own concurrency design to ensure that the returned error code is used properly.

Precondition
NCL is initialized.
Postcondition
The NCL error code is cleared.
Returns
The error code (See the NclErrorCode enum).
