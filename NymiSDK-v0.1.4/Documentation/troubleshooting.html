Troubleshooting Tips

When NCL Fails to Initialize

If nclInit returns NCL_FALSE indicating that the Nymi Communication Library has failed to initialize, then try the following actions:

  1. Ensure that there is not already an instance of your program running.
  2. Ensure that there are no other Nymi-Enabled Applications running.
  3. If you are on Windows, ensure that the BlueGiga dongle is plugged in. You may want to unplug it and replug it.
  4. If you are developing for iOS, ensure that Bluetooth is enabled and functioning properly. You could use a Bluetooth sniffer app like LightBlue.




When a Nymi Disconnects

nclAgree, nclValidate, and nclGlobalSign will all set up a connection to the Nymi. Most NCL functions can only be called when the Nymi is connected. While disconnection is expected after calling functions such as nclProvision, sometimes a Nymi may disconnect unexpectedly.

The .disconnection.reason field of an NCL_EVENT_DISCONNECTION NclEvent tells you the possible reason for the disconnection. View the NclDisconnectionReason enum definition for a list of these reasons. Note that different platforms discern disconnection reasons differently, so this field should be considered "bonus information, if available."




When nclAgree Fails

There is a limit to how many provision keys can be stored on a Nymi. When the limit has been reached, nclAgree will return NCL_FALSE to indicate that no more provisions can be made. On the Developer Unit, you can flip the clasp switch back and forth to clear all the stored provision keys. On the Consumer Unit, a factory reset will clear all the provision keys.




Hardware Debugging

The Nymi Developer Unit User Guide provides information on how to use the Nymi hardware. It also includes tips for troubleshooting.
