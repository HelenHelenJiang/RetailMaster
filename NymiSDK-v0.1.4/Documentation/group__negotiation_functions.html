NclBool nclClearScannedNymis ( )

The NCL distinguishes Nymis based on their MAC address, this allows a Nymi that has been scanned twice to be labeled with the same handle. However, Nymis, when not connected, switch their MAC addresses periodically so that they can't be tracked. When this happens, the corresponding Nymi handle will no longer work but the information about the Nymi is still stored by the NCL. Since a large internal list of Nymis may slow down performance or eventually cause a crash, you may want to clear the internal list of the Nymis. All Nymi handles assigned before a call to this function become invalid when it is called. For this reason, you may only call this function when no Nymi is connected.

Precondition
NCL is initialized and no Nymi is connected.
Postcondition
The internal list of Nymis is cleared.
Returns
NCL_TRUE On success.
NCL_FALSE On failure.

unsigned  intervalMax,
unsigned  timeout,
unsigned  latency 
)

This function will affect the next connection you make. With Core Bluetooth (iOS and Mac), you do not have control over these parameters. See the Bluetooth Accessory Design Guidelines for Apple products. If the vocabulary used here is unfamiliar, you may want to look at a BLE resource.

Precondition
NCL is initialized.
Postcondition
The next connection will use the specified parameters.
Parameters
[in]intervalMinMinimum connection interval. Range is 7.5ms to 4s. Where applicable, default is 10ms. In units of 1.25ms.
[in]intervalMaxMaximum connection interval. Must be greater than intervalMin. Range is 7.5ms to 4s. Where applicable, default is 625ms. In units of 1.25ms.
[in]timeoutSupervision timeout. Must be greater than connection interval. Range: 100ms to 32s. Where applicable, default is 3s. In units of 10ms.
[in]latencySlave latency. Range is 0 to 500. Where applicable, default is 10.
Returns
NCL_TRUE Parameters accepted.
NCL_FALSE One or more parameters not guaranteed to be as specified.
NclBool nclAgree ( int  nymiHandle)

When Nymis are provisioning, they are indistinguishable, which is important in ensuring they remain untrackable. However, it also makes it harder for a user to be sure the correct Nymi and NEA have found each other when provisioning. This is what agreement solves. An LED pattern is derived and the Nymi will display it. You are responsible for making your NEA display it as well. The user will check to make sure both devices are displaying the same pattern, then allow the provisioning process to complete. (See Typical Application Workflow.)

Precondition
NCL is initialized. The specified Nymi has been recently discovered and is not connected.
Postcondition
A connection with the specified Nymi is established, its command channel is taken, and NCL_EVENT_AGREEMENT will be triggered if it agrees, at which point its command channel is free.
Parameters
[in]nymiHandleThe handle of the Nymi you want to agree with.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.
NclBool nclProvision ( int  nymiHandle)

Request to provision a Nymi. An NCL_EVENT_PROVISION NclEvent is triggered if provisioning succeeds. The provision event will contain an NclProvision. You should store this provision locally. Erasing it later is sufficient for effectively revoking a Nymi.

Precondition
NCL is initialized. The specified Nymi has agreed, has not disconnected since, and has a free command channel.
Postcondition
The Nymi's command channel is taken, and an NCL_EVENT_PROVISION will be triggered if the specified Nymi is provisioned, at which point its command channel is free.
Parameters
[in]nymiHandleThe handle of the Nymi you want to create a provision for.
Returns
NCL_TRUE On successful initiation.
NCL_FALSE On failed initiation.

NclBool nclGetConnected ( int *  nymiHandles,
unsigned *  nymiHandlesSize 
)

This function provides the handles of all the Nymis that are currently connected.

Precondition
NCL is initialized.
Postcondition
The handles of all (or as many fit) connected Nymis are stored in nymiHandles, and the number of elements written to nymiHandles is stored in nymiHandlesSize.
Parameters
[out]nymiHandlesAn array in which to put connected Nymi handles.
[in,out]nymiHandlesSizeUpon calling, a pointer to a variable containing the maximum number of Nymi handles that can be stored in the nymiHandles array. After returning, the variable pointed to will be the number of Nymi handles written into the Nymi handle array.
Returns
NCL_TRUE On success.
NCL_FALSE On failure.

Postcondition
