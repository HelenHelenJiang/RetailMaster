Adding the NCL to a Project

View instructions for:

Visual Studio 2012
Eclipse Android
Xcode (OS X)
Xcode (iOS)

Visual Studio 2012

  1. Get ncl.h, ncl.lib, and ncl.dll from the SDK and put them somewhere on your device. You could put them in a common location for all projects to use, or somewhere for just this project.
  2. Open Visual Studio 2012. Open your existing project or create a new one.
  3. In the Solution Explorer, right-click on the project you want to talk with Nymis and select Properties.
  4. On the window that pops up, click on the drop-down menu at the top labelled Configuration and select "All Configurations". Do this for the other platforms if you have multiple.
  5. On the navigation pane on the left, expand Configuration Properties.
  6. Open VC++ Directories. In the next steps, try to use relative paths as opposed to absolute ones. $(ProjectDir) is useful for indicating the current project directory.
  7. Add the directory in which you put ncl.h to Include Directories.
  8. Add the directory in which you put ncl.lib to Library Directories.
  9. Expand Linker. Click on Input. Add ncl.lib to Additional Dependencies.


  10. In the same pop-up window, expand Debugging.
  11. In Environment, add the directory in which you put ncl.dll to PATH. For example,
    PATH=/nclDllDirectory;PATH%
    Note that the environment will be different in deployment; ncl.dll should be put next to the executable or installed into a common DLL location.

Ecplise (Android)

Ecplise (Android)

  1. Get nclSDK.jar and armeabi folder from the SDK and put them somewhere on your device. You could put them in a common location for all projects to use, or somewhere for just this project.
  2. Open Ecplise. Open your existing project or create a new one.
  3. Go to your project Properties. Select 'Java Build Path' on the left of the window. Select the Libraries tab and 'Add External JARS..'. Find and add nclSDK.jar
  4. Drag the armeabi folder into your projects 'libs' folder
  5. In your project Manifest, add Internet permission <uses-permission android:name="android.permission.INTERNET" />

Xcode (OS X)

  1. Get ncl.h and libNCL.dylib from the SDK and put them somewhere on your device. You could put them in a common location for all projects to use, or somewhere for just this project.
  2. Open Xcode. Open your existing project or create a new one.
  3. In the Build Settings of the project or target you want to talk with Nymis, add the directory in which you put ncl.h to Header Search Paths. For example, if ncl.h is next to the theProject.xcodeproj, you could add "." (the current directory).
  4. Drag libNCL.dylib into your project. If you want to avoid absolute paths, make sure the paths added to your target/project are relative.
  5. In the Build Phases of the target you want to talk with Nymis, in Copy Files, select Executables as a Destination, clear the subpath, make sure "Copy only when installing" is unselected and add libNCL.dylib.
  6. In the Build Settings of the project or target you want to talk with Nymis, add "." (the current directory) to Runpath Search Paths . This and the previous step can be modified to change how the dylib is used.

Xcode (iOS)

  1. Get ncl.h and libNCLiOS.a from the SDK and put them somewhere on your device. You could put them in a common location for all projects to use, or somewhere for just this project.
  2. Open Xcode. Open your existing project or create a new one.
  3. In the Build Settings of the project or target you want to talk with Nymis, add the directory in which you put ncl.h to Header Search Paths. For example, if ncl.h is next to the theProject.xcodeproj, you could add "." (the current directory).
  4. Drag libNCLiOS.a into your project. If you want to avoid absolute paths, make sure the paths added to your target/project are relative.
  5. Add the Core Bluetooth framework to your project.
