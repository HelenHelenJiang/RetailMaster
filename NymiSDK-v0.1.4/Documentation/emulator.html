The Nymulator

The Nymi Emulator (or Nymulator) allows developers like you to start writing Nymi-Enabled Applications (NEAs) without a physical Nymi. The Nymulator is a software that pretends to be a Nymi in developer mode, which means it is always in an authenticated state.

Only one session of the Nymulator can be open at any one time.




The Nymulator Interface

Screenshot of the Nymulator

Managing Nymis

You can manage your simulated Nymis using the left panel. There are 3 round buttons:

  • New: create a new Nymi
  • Load: load a list of previously saved Nymis
  • Save: save the current list of Nymis

The Nymulator allows you to save the data associated with simulated Nymis, including: provisions, symmetric keys created with nclCreateSk, and private signing keys created with nclCreateSigKeyPair and nclCreateGlobalSigKeyPair.

After highlighting a Nymi green by clicking on it, you can now either activate the Nymi by clicking the arrow, or delete it from the list by clicking the minus sign. Activating a Nymi is a Nymulator-specific term, it means you can now put it into provisioning mode, or disconnect it using the buttons on the right panel. The Nymi Log and the diagram will also reflect the activated Nymi. Only one Nymi can be activated at any one time, and it will be marked with a red outline.

Please note that activated is not the same as authenticated. Because the Nymulator simulates Nymis in developer mode, all Nymis in it are always authenticated. Activated means that you can interact with and read the log from that specific Nymi.

Interacting with the Active Nymi

Once you've activate a Nymi from your list, the log should now tell you that it is advertising.

If you wish to put it in provisioning mode, click on the "provision" button. You should now see an "inside-out" pulsing LED pattern on the diagram in the center panel. After a provision has been established, the diagram will display a unique LED pattern, and provision is completed when nclAgree is called because the displayed pattern matches the pattern sent to the NEA. Please refer to Typical Application Workflow for more information on this process.

If you click on the "disconnect" button during provisioning, you would stop the process. If you click on it at another time, then you would break any existing connections.

In the center panel, there is a sliding bar labelled "RSSI Value", changing the RSSI mimics changing the distance between the Nymi and the device that is running the NEA. The lower the value, the further away the Nymi appears to be.




Using the Nymulator with Your App


Running a Nymulator on the Same Device as the NEA

Ensure that you launch the Nymulator before launching your app. Your app should be able to connect to and send commands to the Nymulator as if it were a real Nymi.

Running a Nymulator on a Different Device than the NEA

Since the Nymulator is currently only available as a desktop application, you may wish to use it with a NEA that is running on another device, such as a phone. To do this, ensure that the device that the Nymulator is running on is on the same network as the device that the NEA is running on. Then prior to calling nclInit, call nclSetIpAndPort and set the IP address to that of the computer that is running to Nymulator.

If nclSetIpAndPort is not called, the default IP address is 127.0.0.1.

Please also note that if you are using the Nymulator, you should not change the port parameter. The Nymulator needs to use port 9089. It will also give an error when that port is taken up already by another application.
